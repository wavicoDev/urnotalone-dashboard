>
> 버전 0.1 · 2026.05

## 1. 웹앱의 목표

URNOTALONE은 아동(7~12세)이 AI 캐릭터 "도리"와 대화하며 매일 비침습적으로 정서 신호를 남기는 서비스다. 이 서비스가 의미를 가지려면 **아동의 신호가 보호자와 교사에게 적절히 닿아야 한다**. 본 웹앱은 그 "닿음"의 인터페이스다.

핵심 목표는 세 가지다.

첫째, **신호의 가시화**다. 아동이 도리와 나눈 감정·활동의 흐름을 보호자와 교사가 한눈에 파악할 수 있어야 한다. 단, 대화 내용 자체가 아닌 추출된 신호(감정 분포, 활동 누적, 위기 수준, 어휘 확장 곡선)만 노출한다. 아동의 사생활은 절대 보장한다.

둘째, **위기의 즉시 전달**이다. SafetyGate FSM이 Lv.2 이상의 위기 신호를 감지한 순간, 보호자 또는 교사에게 알림이 닿아야 한다. 이 알림은 구체적 발화가 아닌 "오늘 친구 관계의 어려움을 표현했어요" 수준의 추상적 신호로 전달되며, 보호자가 직접 자연스럽게 말을 거는 가교 역할을 한다.

셋째, **양육·교육 파트너십**이다. 보호자는 주간 코칭 카드를, 교사는 학급 단위 리포트를 받는다. 도리는 일상 자기조절 훈련을 분담함으로써 상담 자원의 절대 부족을 메우는 선행 보완재다.

대상 사용자는 두 집단이다. 보호자(주로 부모)는 자녀 1명을 중심으로 한 1:1 관계의 정서 가교다. 교사는 학급 전체(20~30명)를 책임지는 1:N 관계의 조기 발견자다. 두 집단은 같은 데이터를 보지만 관점과 권한이 다르므로, 대시보드는 명확히 분리한다.

## 2. 메인 기능

### 2.1 공통 기능

- **인증·역할 분기**: 이메일+비밀번호 로그인. 가입 시 보호자/교사 역할 선택. 보호자는 자녀 등록, 교사는 학교·학급 등록.
- **실시간 알림**: 위기 신호, 주간 리포트 발행, 도리 활동 완료 등 이벤트 알림. 헤더 종 모양 아이콘 + 푸시(Stage 3).
- **알림 메일 발송 로그**: 시스템이 자동 발송한 보호자 알림 메일의 발송 이력을 확인.
- **개인정보·동의 관리**: 보호자 동의 철회, 데이터 삭제 요청, 데이터 거버넌스 정책 확인.

### 2.2 부모용 대시보드 핵심 기능

**홈 (오늘의 마음)**
오늘 자녀가 도리와 나눈 대화의 핵심 신호를 1개 카드로 요약한다. 감정 이모지 1개, 활동 1개, 코칭 제안 1줄. "오늘 저녁 자연스럽게 이야기 나눠 보세요" 같은 행동 제안.

**감정 타임라인**
최근 7일/30일/90일의 감정 분포 라인 차트. 5단계 이모지 비율 + 위기 수위 마커. 클릭 시 해당 일자의 활동 카드(역할극·감정기록·감사일기 등 수행 여부)를 펼침.

**성장 마일스톤**
감정 어휘 확장 곡선(TTR), 감사 일기 연속 일수, 자율적 인지 재구성 시도 빈도 등 성장 지표. "지난주보다 감정 단어 3개 늘었어요" 같은 변화 메시지.

**주간 코칭 카드**
매주 일요일 자동 발행. 이번 주 자녀의 핵심 신호 1~2개 + 대화법 코칭 1개 + 함께 해볼 수 있는 작은 활동 제안 1개. PDF로 다운로드 가능.

**위기 알림 센터**
SafetyGate가 Lv.2~3을 감지한 이력. 감지 시각, 카테고리(따돌림/언어폭력/자해/학대), 발송 알림 본문(요약본). 클릭 시 권장 대응 가이드 + 외부 자원(Wee센터, 1393, 117 등) 연결.

**자녀 프로필·설정**
이름, 학년, 친구 이름(도리가 기억할), 부모 호칭, 알림 채널(이메일/푸시), 알림 민감도(Lv.2부터/Lv.3만).

### 2.3 교사용 대시보드 핵심 기능

**학급 개요**
우리 반 학생 N명의 출석·정서 상태를 한눈에 보는 그리드. 학생 카드 1개당 오늘의 5-emoji 응답 + 위기 신호 여부 + 도리 활동 수행 여부. 위기 신호 학생은 카드 좌측에 빨간 띠.

**학생 상세**
개별 학생 클릭 시 진입. 부모 대시보드의 감정 타임라인과 동일하되, 발화 원문은 보이지 않고 카테고리·요약·수위만 보임. 교사 메모 기능(이 학생은 ADHD 의심, 가정 상담 진행 중 등).

**위기 신호 큐**
오늘 우리 반에서 발생한 Lv.2 이상 신호를 시각 순으로 나열. 각 카드에 "확인·후속 조치 기록" 버튼. 조치 기록(상담실 연결, 학부모 연락 등)은 누적되어 학생 상세에 표시.

**주간 학급 리포트**
매주 월요일 자동 발행. 학급 평균 감정 분포, 위기 신호 발생 건수, 활동 참여율, 주의 학생 목록(익명 코드 또는 동의 시 실명). 학교 단위 보고용 PDF 출력.

**상담 노트 (역할극 카드 추가)**
도리의 역할극 활동에 "우리 반에 필요한 상황 카드"를 추가하는 기능. 예: 새 전학생 적응, 시험 후 다툼. 추가한 카드는 우리 반 학생들에게만 노출된다.

**학교·학급 설정**
학교명, 학년·반, 학생 명단(보호자 동의 받은 학생만), 교사 권한(담임/상담교사), 알림 채널.

### 2.4 데이터 정책 (양 대시보드 공통)

- 대화 원문은 절대 노출하지 않는다. 추출된 신호(감정 라벨, 활동 카테고리, 수위, 통계)만 노출한다.
- 위기 알림 본문은 "친구 관계의 어려움", "정서 표현이 평소와 달라요" 수준의 추상적 메시지로 통일한다.
- 보호자 동의가 없는 학생은 교사 대시보드에 익명 코드(학생A, 학생B)로만 표시된다.

## 3. 유저 흐름

### 3.1 부모 — 일상 흐름 (Day 7 시나리오 기반)

저녁 8시. 어머니의 휴대폰에 알림이 도착한다. "수민이의 마음 신호 알림"이라는 제목이다. 어머니는 알림을 탭하고 부모 대시보드의 위기 알림 센터로 진입한다. 본문은 "오늘 수민이가 도리와의 대화에서 친구 관계의 어려움을 표현했어요. 오늘 저녁 자연스럽게 이야기 나눠 주세요"라고 적혀 있다. 구체 대화 내용은 없다.

어머니는 알림 아래의 권장 대응 가이드를 읽는다. "평가·강요 없이 들어주세요", "비밀로 하겠다는 약속은 피해주세요" 같은 4종 금기를 안내받는다. 어머니는 그날 저녁 자연스럽게 말을 걸고, 수민이는 처음으로 친구 이야기를 털어놓는다.

다음 날 아침, 어머니는 감정 타임라인을 열어 최근 7일 흐름을 본다. 이모지가 점차 가라앉는 패턴이 보인다. 일요일 오전, 주간 코칭 카드가 발행된다. "이번 주는 친구 관계가 어려웠던 한 주였습니다. 다음 주 등교 전에 작은 응원 한마디를 건네보세요" 같은 제안.

### 3.2 교사 — 일상 흐름

월요일 아침 8시 30분. 교사가 출근해 학급 개요를 연다. 1교시 전 5분, 우리 반 23명의 카드 그리드가 펼쳐진다. 22명은 노란·초록 이모지, 1명(수민)이 회색 이모지 + 좌측 빨간 띠. 위기 신호 큐를 확인하니 "Lv.2 따돌림 카테고리, 금요일 감지" 기록이 있다.

교사는 수민의 학생 상세를 열어 최근 7일 흐름을 본다. 구체 발화는 보이지 않지만 감정 라벨이 "외로움 → 슬픔 → 분노"로 흘러온 패턴을 확인한다. 교사는 1교시 후 쉬는시간에 수민을 자연스럽게 부르고, "확인·후속 조치 기록"에 "쉬는시간 짧은 대화, 본인 답변 회피, 상담실 연결 예정"을 남긴다.

수요일, 학교 차원의 보고가 필요해 주간 학급 리포트의 지난주 분량을 PDF로 출력해 교감에게 제출한다. 보호자 동의 미수령 학생은 익명 코드로 처리되어 있다.

목요일, 교사는 상담 노트에서 "전학생 첫 주" 상황 카드를 추가한다. 다음 주 새 전학생이 합류하기 때문이다. 이 카드는 우리 반 학생들이 도리와 역할극을 할 때 추천된다.

### 3.3 가입·초기 설정 흐름

**보호자**: 이메일 회원가입 → 보호자 역할 선택 → 약관·데이터 동의 → 자녀 추가(이름·생년월·학년) → 자녀의 도리 앱과 페어링(QR 또는 코드) → 알림 채널·민감도 설정 → 홈 진입.

**교사**: 이메일 회원가입 → 교사 역할 선택 → 학교 인증(교사 인증 코드 또는 학교 관리자 승인) → 학년·반 설정 → 학생 명단 업로드(CSV 또는 수기) → 보호자에게 동의 요청 메일 자동 발송 → 동의 수령 학생부터 실명·신호 표시.

### 3.4 위기 신호 발생 시 흐름 (시스템 관점)

도리(아동 앱) → SafetyGate FSM이 발화에서 위기 키워드 감지 → 수위 판정 (Lv.1/2/3) → Lv.2 이상이면 알림 메일 발송 + 대시보드 위기 알림 센터에 카드 추가 + 헤더 종 알림 → 보호자/교사가 확인 → 후속 조치 기록 → Lv.3는 추가로 외부 자원(1393 등) 안내 화면 강제 노출.

## 5. 사이트별 UI 구성

### 5.1 부모용 UI

**/home (오늘의 마음)**

상단 헤더는 좌측 도리 캐릭터 + "수민이의 오늘" 타이틀, 우측 종 알림 아이콘과 프로필 아바타. 본문 중앙에 큰 카드 1개. 카드 안에는 오늘의 대표 이모지(120px), 오늘 도리와 나눈 활동 라벨(예: "감사 일기 · 친구"), 코칭 제안 1줄. 카드 하단에 "타임라인 보기" "코칭 카드 보기" 두 개의 버튼. 카드 아래에 작은 위젯 3개 (이번 주 감정 분포 미니 라인, 마일스톤 진행률, 최근 알림 1건).

**/timeline (감정 타임라인)**

상단에 7d/30d/90d 토글. 본문에 가로축 날짜·세로축 감정 라인 차트. 위기 마커는 빨간 점으로 표시. 차트 하단에 5단계 이모지 분포 도넛(같은 기간). 좌측 패널에는 일별 활동 카드 리스트 (날짜·활동·이모지). 카드 클릭 시 우측에 그날의 활동 상세 패널 펼침 (대화 원문 없음, 활동 카테고리·수위만).

**/growth (성장 마일스톤)**

상단에 3개의 큰 지표 카드 (감정 어휘 수·감사 일기 연속·자율 재구성 시도). 각 카드는 큰 숫자 + 전주 대비 변화량 + 미니 스파크라인. 본문에 마일스톤 타임라인 (성취 배지 형식: "감정 단어 30개 돌파", "감사 일기 7일 연속" 등).

**/coaching (주간 코칭 카드)**

상단에 최신 카드 (이번 주). 카드 안에는 주차 표시, 핵심 신호 1~2줄, 대화법 코칭, 작은 활동 제안. 우측에 "PDF 다운로드" 버튼. 본문 하단에 과거 카드 리스트 (썸네일 + 주차 + 핵심 신호 한 줄).

**/alerts (위기 알림 센터)**

상단에 필터 (전체/Lv.2/Lv.3, 최근 30일). 본문에 알림 카드 리스트, 각 카드에 시각·카테고리 뱃지·요약 본문·확인 여부. 미확인은 좌측에 노란 띠, Lv.3는 빨간 띠. 카드 클릭 시 상세 페이지로 이동.

**/alerts/:alertId (알림 상세)**

상단에 알림 헤더 (시각·카테고리·수위). 본문에 알림 본문, 권장 대응 가이드 (4종 금기 안내·말 거는 방법 3가지), 외부 자원 링크 (Wee센터·1393·117 버튼). 하단에 "확인 완료" 버튼 + 메모 입력란.

**/child (자녀 프로필)**

탭 3개 (기본 정보·친구 이름·알림 설정). 친구 이름 탭에서는 도리가 기억할 친구 이름을 추가·삭제 (예: 민지·서연). 알림 설정에서는 채널(이메일/푸시) 토글 + 민감도 슬라이더 (Lv.2부터 받기 / Lv.3만 받기).

### 5.2 교사용 UI

**/class (학급 개요)**

상단 헤더에 학교·학급명·날짜·우리반 N명 카운트. 본문에 학생 카드 그리드 (4~6열 반응형). 각 학생 카드는 정사각형 (학생 이름 또는 익명 코드, 오늘의 이모지 60px, 활동 수행 점 3개 표시). 위기 신호 학생은 카드 좌측에 빨간 띠 + 우상단 작은 경고 아이콘. 카드 클릭 시 학생 상세로 이동. 우측 사이드바에 오늘의 요약 (반 평균 이모지·활동 참여율·위기 신호 N건).

**/class/students/:id (학생 상세)**

부모용 타임라인 페이지와 동일한 차트 구조. 단, 상단에 "교사 권한 정보 표시 중" 안내 배너 + 학생 메모 탭 추가. 메모 탭은 교사 본인만 보이는 사적 노트. 본문 하단에 "최근 후속 조치 이력" 리스트 (상담실 연결·학부모 연락 등).

**/alerts (위기 신호 큐)**

학급 전체의 위기 신호를 시각 순으로 나열. 각 카드에 학생명(또는 코드)·시각·카테고리·수위·요약. 미확인 카드는 상단 고정, 확인 카드는 회색 처리. 카드 우측에 "확인·기록" 버튼 (인라인 드롭다운으로 후속 조치 입력).

**/report (주간 학급 리포트)**

상단에 주차 선택 드롭다운. 본문에 4개 섹션. ① 학급 평균 감정 분포 (도넛 차트), ② 위기 신호 발생 통계 (카테고리별 막대), ③ 활동 참여율 (활동 6종별 진행률 바), ④ 주의 학생 목록 (실명 또는 익명 코드 + 핵심 신호 1줄). 우측 상단에 "PDF 출력" 버튼.

**/cards (상담 노트)**

상단에 카드 카테고리 탭 (또래/학업/가족/자기 표현/우리 반 추가). 본문에 카드 그리드. 우리 반 추가 탭에서 "+" 버튼으로 신규 카드 추가. 카드 추가 폼은 카테고리 선택·상황 제목·짧은 설명·역할(친구/선생님/부모) 선택.

**/cards/new (카드 추가 폼)**

단일 페이지 폼. 카테고리·제목·상황 묘사(2~3문장)·역할·연령대(저학년/고학년) 입력. 저장 시 우리 반에만 노출되는 카드가 도리 시스템에 동기화.

### 5.3 공통 UI 요소

**헤더**: 좌측 로고(도리 캐릭터 작게)·중앙 메뉴 탭·우측 알림 종·프로필 아바타. 알림 종 클릭 시 드롭다운으로 최근 5건 미리보기.

**사이드 네비게이션**: 데스크탑은 좌측 고정 사이드바 (아이콘 + 라벨), 모바일은 하단 탭바.

**알림 토스트**: 우상단에서 슬라이드 인, 위기 알림은 빨간 좌측 띠 + 진동(Stage 3).

**빈 상태 (Empty State)**: 데이터가 없는 화면에서 도리 일러스트 + "아직 신호가 없어요. 도리가 친구가 되는 중이에요" 같은 안내.

## 7. 디자인 가이드

### 7.1 톤 & 매너

대상은 보호자와 교사이지만, 이 서비스의 주인공은 아동이다. 그래서 대시보드는 두 가지 톤을 동시에 가진다. 하나는 **차분한 신뢰감** (의료·교육 도메인의 진중함), 다른 하나는 **따뜻한 친근감** (도리 캐릭터의 정서).

전반적으로 부드러운 색상, 둥근 모서리, 여백이 넉넉한 레이아웃을 사용한다. 위기·경고 영역에서만 채도가 높은 색을 쓴다. 그래프와 차트는 명료하되, 차가운 데이터 분석 도구처럼 보이지 않도록 부드러운 곡선과 둥근 점을 사용한다.

### 7.2 컬러 팔레트

**기본 (도리 정서 케어 팔레트)**

- Primary `#5B8DEF` — 신뢰의 블루. 주요 버튼·링크·강조
- Primary Soft `#E8F0FE` — Primary의 배경 톤. 카드 배경·호버
- Accent Warm `#FFB46B` — 도리 캐릭터의 따뜻한 오렌지. 마일스톤·긍정 신호
- Accent Mint `#7BD3B5` — 평온의 민트. 정상 상태·완료
- Neutral 900 `#1F2937` — 본문 텍스트
- Neutral 700 `#4B5563` — 보조 텍스트
- Neutral 400 `#9CA3AF` — 비활성·플레이스홀더
- Neutral 100 `#F3F4F6` — 카드 구분선·배경
- Background `#FAFBFC` — 페이지 배경
- Surface `#FFFFFF` — 카드 표면

**위기·경고**

- Alert Lv.1 `#FCD34D` — 노란빛, 주의 (불편)
- Alert Lv.2 `#FB923C` — 주황, 경고 (심각)
- Alert Lv.3 `#EF4444` — 빨강, 위급

**감정 5단계 (아동과 동일)**

- Very Sad `#A78BFA` (보라)
- Sad `#60A5FA` (파랑)
- Neutral `#9CA3AF` (회색)
- Happy `#FBBF24` (노랑)
- Very Happy `#34D399` (초록)

### 7.3 타이포그래피

한국어 본문은 `Pretendard` 또는 시스템 한글 폰트. 영문은 `Inter` 또는 `Noto Sans`. 도리 캐릭터 메시지나 아동 관련 카드 제목에는 살짝 둥근 느낌의 `Pretendard Medium`을 사용해 친근감을 준다.

- H1 `28px / 700 / -0.02em` — 페이지 타이틀
- H2 `22px / 600` — 섹션 타이틀
- H3 `18px / 600` — 카드 타이틀
- Body `15px / 400 / line-height 1.6` — 본문
- Caption `13px / 400` — 보조 정보
- Number `40px / 700` — 큰 지표 숫자 (성장 마일스톤 등)

### 7.4 레이아웃·간격

- 기본 그리드: 12 컬럼, 데스크탑 max-width 1200px, 모바일은 단일 컬럼
- 간격 스케일: 4 / 8 / 12 / 16 / 24 / 32 / 48 / 64px
- 카드 패딩: 24px (데스크탑) / 16px (모바일)
- 카드 모서리: 16px radius (큰 카드) / 12px (중간) / 8px (작은 버튼·태그)
- 그림자: `0 2px 8px rgba(31, 41, 55, 0.06)` (기본 카드) / `0 8px 24px rgba(31, 41, 55, 0.10)` (호버·강조)

### 7.5 컴포넌트 원칙

**버튼**

- Primary: Primary 컬러 배경, 흰 글자, 14px padding-vertical
- Secondary: 흰 배경, Primary 테두리·글자
- Ghost: 배경 없음, Neutral 700 글자, 호버 시 Neutral 100 배경
- Danger: Alert Lv.3 배경, 흰 글자 (위기 알림 확인 등 신중 행동)

**카드**

기본 흰 표면 + 둥근 모서리 + 부드러운 그림자. 호버 시 그림자 강화. 위기 카드는 좌측에 4px 두께의 색띠 (Lv.2 주황·Lv.3 빨강).

**차트**

라인 차트는 곡선(monotone), 점은 둥근 5px. 도넛 차트는 안쪽에 큰 숫자 배치. 색은 위 팔레트의 감정 5단계 컬러 사용.

**아이콘**

`lucide-react` 또는 동급의 둥근 라인 아이콘 세트. 16/20/24px 통일.

**캐릭터**

도리의 표정 6종 PNG를 빈 상태·랜딩·완료 화면 등에 활용. 차분한 정보 위주 화면에서는 캐릭터 사용을 자제하고, 정서 환기가 필요한 지점에만 등장시킨다.

### 7.6 인터랙션 가이드

- 페이지 전환은 짧은 페이드(150ms)
- 카드 호버는 그림자 변화 + 1~2px 위로 이동
- 위기 알림 토스트는 살짝 흔들리는 진입 모션 (주의 환기)
- 데이터 로딩은 스켈레톤 UI 사용, 빈 차트보다 회색 라인 형태 우선

### 7.7 접근성

- 본문 텍스트 대비비 4.5:1 이상 유지
- 모든 버튼·링크는 키보드 포커스 가능 (2px Primary 외곽선)
- 색만으로 정보를 전달하지 않음 (위기 수위는 색 + 아이콘 + 라벨 함께)
- 폼 입력 오류는 빨간색 + 텍스트 메시지 동시 표기

### 7.8 톤 라이팅 원칙

- 부모 대시보드: 따뜻하지만 평가하지 않는 톤. "수민이가 오늘 외로움을 표현했어요" (O), "수민이가 안 좋아요" (X)
- 교사 대시보드: 사실 위주의 전문적 톤. "Lv.2 따돌림 카테고리 신호 감지" (O)
- 빈 상태: 도리 화자 톤. "아직 신호가 없어요. 도리가 친구가 되는 중이에요"
- 위기 알림: 짧고 명료하며 행동 지시. "오늘 저녁 자연스럽게 이야기 나눠 주세요"

## 8. 개발 스테이지

각 스테이지는 **그 자체로 동작 가능한 제품**이어야 한다. 다음 스테이지가 추가되어도 이전 스테이지의 화면·흐름은 그대로 보존된다.

### Stage 1 — 프론트엔드 단독 (Backendless)

**목표**: 백엔드 없이도 부모·교사 두 대시보드의 모든 화면을 시연 가능. 데이터는 클라이언트 측 mock 데이터와 `localStorage`로 구성.

**구현 범위**

- 모든 라우트 (`/parent/*`, `/teacher/*`, 공통 페이지) UI 완성
- 상태 관리는 React Context + `localStorage` 영속화
- mock 데이터셋: 가상 자녀 2명(수민·준호) + 가상 학급 1개(23명) + 14일치 EmotionLog · CrisisAlert · WeeklyReport
- 모든 차트는 mock 데이터로 실제 렌더링 (라인·도넛·바)
- 로그인 화면은 더미 로그인 (이메일·비밀번호 입력 후 로컬 토큰 저장, 검증 없음)
- 위기 알림은 mock 데이터로 카드 표시, 확인·후속 조치는 `localStorage`에 누적
- 주간 코칭 카드 PDF 다운로드는 클라이언트 측 PDF 생성 (jsPDF 등)
- 도리 캐릭터 표정 PNG, 디자인 시스템 컴포넌트 라이브러리 완성
- 시연 시나리오: Day 7 보호자 알림 흐름, 교사 학급 개요 + 위기 큐 흐름 두 가지를 실제 화면으로 보여줄 수 있다

**제외**

- 실제 인증
- 서버 API 호출
- 도리 앱(아동 앱)과의 실제 연동
- 알림 메일 실제 발송
- 멀티 디바이스 동기화

**완료 기준**: 외부 네트워크가 차단된 환경에서 두 페르소나의 시연 시나리오가 처음부터 끝까지 진행된다.

### Stage 2 — REST API 백엔드 통합

**목표**: 모든 데이터가 서버에서 관리되며 인증·동의·알림이 실제로 작동. 단, 실시간 푸시는 제외.

**구현 범위**

- 인증: 이메일+비밀번호, JWT 발급, 세션 관리, 보호자/교사 역할 분기
- REST API 엔드포인트 (모든 데이터 명세의 리소스에 대한 CRUD)
- 도리 앱(아동 앱)이 EmotionLog를 POST하면 대시보드에 반영 (도리 앱 자체는 본 기획 범위 밖, 단 webhook 또는 토큰 기반 API로 받음)
- SafetyGate 위기 감지 결과를 받는 webhook: `POST /api/crisis`
- 알림 메일 실제 발송 (Resend 등 메일 서비스)
- 주간 리포트 자동 생성 (서버 사이드 cron, 매주 발행)
- 보호자 동의 워크플로우: 교사가 학생 추가 → 보호자에게 동의 요청 메일 → 보호자 클릭 → 서버 기록 → 교사 대시보드에 실명 표시
- PDF 생성을 서버에서 처리하고 다운로드 링크 제공
- Stage 1에서 만든 mock 데이터는 Seed 데이터로 변환되어 개발·시연용 계정에 자동 주입

**페이지·기능 동작**

- 위기 알림 큐는 페이지 진입 시 또는 새로고침 시 갱신 (실시간 X)
- 알림 메일은 서버에서 발송, 대시보드의 알림 센터에 카드 추가
- 학생 명단 CSV 업로드, 동의 워크플로우, 후속 조치 기록 모두 서버 영속

**완료 기준**: 두 대시보드가 서버 데이터를 기반으로 완전히 동작하고, 보호자·교사 다중 사용자 환경에서 데이터가 격리·공유된다. 위기 알림은 메일로 도달하고 대시보드에는 새로고침으로 반영된다.

### Stage 3 — 실시간 기능

**목표**: 위기 신호와 학생 활동이 새로고침 없이 즉시 대시보드에 반영. 푸시 알림이 디바이스에 도달.

**구현 범위**

- WebSocket 또는 SSE 채널: 위기 알림, 새 EmotionLog, 학급 개요 카드 업데이트를 실시간 푸시
- 웹 푸시 알림 (Web Push API): 부모는 자녀 위기 신호, 교사는 학급 위기 신호를 디바이스 알림으로 수신
- 학급 개요 그리드: 학생이 도리와 활동 완료 시 카드의 이모지·점이 즉시 변화 (애니메이션 페이드)
- 위기 알림 토스트: 다른 페이지에 있어도 우상단 토스트로 즉시 노출 (소리·진동 옵션)
- 학생 상세 페이지 동시 열람 시 다른 교사의 메모 추가가 실시간 반영
- 후속 조치 기록도 실시간 동기화 (동일 학교 교사 간 협업)
- 연결 상태 표시 UI (헤더에 작은 점, 연결됨/재연결 중/오프라인)
- 오프라인 시 큐잉 → 재연결 시 동기화

**완료 기준**: 도리 앱에서 발생한 위기 신호가 5초 이내에 부모·교사 디바이스의 푸시 알림 + 대시보드 토스트로 도달한다. 학급 개요는 새로고침 없이 항상 최신 상태를 유지한다.

## 9. 고려사항

### 9.1 데이터 윤리·법적 고려

본 서비스의 핵심 사용자는 만 14세 미만 아동이다. 한국 개인정보보호법 및 정보통신망법상 만 14세 미만 개인정보 수집은 법정대리인 동의가 필수다. 따라서 보호자 동의는 시스템 진입의 전제 조건이며, 동의 없이는 어떠한 데이터도 수집·노출되지 않는다. 동의 철회 시 30일 이내 모든 데이터를 삭제하는 워크플로우가 필요하다.

교사 대시보드는 추가 동의(`teacher_visibility`)가 별도로 필요하다. 보호자가 "내 자녀의 신호를 교사가 볼 수 있다"에 동의하지 않으면, 해당 학생은 교사 대시보드에 익명 코드로만 표시되며 신호 상세는 노출되지 않는다.

대화 원문은 절대 대시보드에 노출하지 않는다. 이는 단순한 정책이 아니라 시스템 설계 차원의 강제이며, EmotionLog 스키마에 원문 필드 자체를 두지 않는다.

### 9.2 위기 알림의 표현 수위

위기 알림 메일·대시보드 본문은 "친구 관계의 어려움", "정서 표현이 평소와 달라요" 수준의 추상적 표현으로 통일한다. "수민이가 자해를 언급했어요" 같은 직접적 표현은 보호자의 과민 반응을 유발할 수 있고, 아이의 신뢰를 깨뜨릴 위험이 있다. 단, Lv.3(자해·학대) 알림은 보호자에게 외부 자원(1393·117) 즉시 연결 화면을 강제 노출한다.

### 9.3 교사의 권한 범위

교사는 학급 전체를 보지만, 개별 학생의 깊은 정서 영역(예: 가족 갈등)에는 권한이 제한될 수 있어야 한다. 향후 보호자가 카테고리별로 교사 열람을 차등 동의(또래만 OK, 가족은 X)할 수 있는 구조를 데이터 모델 단계에서 열어둔다. 다만 본 기획서의 1차 범위에서는 전체 동의/미동의 이분법으로 단순화한다.

### 9.4 오탐·과탐의 균형

SafetyGate는 룰 기반(Stage 2 시점)이므로 오탐이 발생할 수 있다. "오늘 망했어"가 일상 표현인지 위기 신호인지 룰만으로 구분이 어렵다. 대시보드는 보호자가 "오탐이었어요" 피드백을 남길 수 있는 버튼을 두어 모델 개선 데이터로 활용하되, 그 피드백이 위기 알림 자체를 차단하지 않도록 한다. 즉, 알림은 항상 발송되고 사후 피드백만 받는다 — 안전이 우선이다.

### 9.5 교사 인증

교사 가입은 일반 보호자 가입보다 신중해야 한다. 부적격자가 교사로 가입해 학급 신호를 열람하는 일을 막아야 한다. 가능한 인증 방법은 ① NEIS 연계, ② 학교 관리자(교감·교장) 승인, ③ 교사 인증 코드 (학교가 발급) 등이 있다. Stage 1·2에서는 인증 코드 방식을 가정하고, 실제 운영에서는 NEIS 연계를 검토한다.

### 9.6 다중 보호자

한 자녀에 보호자가 둘 이상일 수 있다 (부·모, 조부모 포함). 데이터 모델에서 `Child.parentId`는 단일이 아닌 `parentUserIds: string[]`로 확장 가능한 구조로 설계한다. Stage 1에서는 단일 보호자로 단순화하되, 스키마는 배열을 염두에 둔다.

### 9.7 도리 앱(아동 앱)과의 경계

본 기획서는 부모용·교사용 **대시보드** 웹앱에 한정한다. 도리 앱(아동이 사용하는 PWA/모바일 앱)은 별도 산출물이다. 두 시스템은 서버 API와 webhook으로 연결되며, 본 기획에서는 도리 앱이 EmotionLog와 CrisisAlert를 정확히 발행한다고 가정한다.

### 9.8 시연 시나리오 우선

이 프로젝트는 공모전·심사를 위한 시연이 1차 목표일 가능성이 높다. 따라서 Stage 1에서 만드는 mock 데이터는 단순히 빈 화면을 채우는 용도가 아니라, **두 개의 완결된 스토리** (Day 7 보호자 시나리오, 교사 학급 개요 시나리오)를 처음부터 끝까지 재현하도록 설계되어야 한다. 데이터는 시간 순으로 일관되게 진행되며, 시연자가 어느 화면에 들어가도 같은 스토리 안에 있다.

### 9.9 디자인 일관성

도리는 아동용 캐릭터이지만, 대시보드는 보호자·교사용이다. 캐릭터를 과하게 노출하면 전문성이 떨어져 보이고, 너무 배제하면 도리와의 정서적 연결이 끊어진다. 캐릭터는 빈 상태·랜딩·완료 화면 등 **정서 환기가 필요한 지점에만** 등장시키고, 데이터·통계 영역에서는 절제한다.
